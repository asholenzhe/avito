# AI-ассистент для улучшения объявлений (тестовое задание, Frontend)

Веб-приложение — личный кабинет продавца с AI-подсказками для описаний и оценки цены. Макеты в Figma: [список и карточки](https://www.figma.com/design/mkeo1cvzQpEqmN3txeDNBo/%D0%9C%D0%B0%D0%BA%D0%B5%D1%82-%D1%82%D0%B5%D1%81%D1%82%D0%BE%D0%B2%D0%BE%D0%B3%D0%BE-%D0%B7%D0%B0%D0%B4%D0%B0%D0%BD%D0%B8%D1%8F-%D1%81%D1%82%D0%B0%D0%B6%D1%91%D1%80%D0%B0%D0%BC?node-id=23-2515&p=f&t=MywyuIX2PE3kiWIS-0), [детали](https://www.figma.com/design/mkeo1cvzQpEqmN3txeDNBo/%D0%9C%D0%B0%D0%BA%D0%B5%D1%82-%D1%82%D0%B5%D1%81%D1%82%D0%BE%D0%B2%D0%BE%D0%B3%D0%BE-%D0%B7%D0%B0%D0%B4%D0%B0%D0%BD%D0%B8%D1%8F-%D1%81%D1%82%D0%B0%D0%B6%D1%91%D1%80%D0%B0%D0%BC?node-id=23-2513&t=MywyuIX2PE3kiWIS-0).

## Стек

- Node.js 20+, React 18+, TypeScript, Vite, react-router-dom, Zustand, Axios, ESLint, Prettier, Vitest
- Бэкенд объявлений: репозиторий включает сервер `server/` с теми же эндпоинтами, что в задании (`GET/PUT /items...`)

## Быстрый старт (локально)

### 1. API объявлений

```bash
cd server
npm install
npm start
```

Сервер слушает порт **8080** (можно задать `PORT`).

### 2. Клиент

В корне проекта:

```bash
npm install
copy .env.example .env   # Windows; на Unix: cp .env.example .env
npm run dev
```

По умолчанию `VITE_API_BASE_URL=/api` — в режиме разработки Vite проксирует запросы на `http://localhost:8080`.

### 3. LLM (обязательно для AI-кнопок)

**Вариант A — Ollama (локально)**

1. Установите [Ollama](https://ollama.com/), выполните `ollama pull llama3` (или другую модель и укажите её в `VITE_OLLAMA_MODEL`).
2. Запустите `ollama serve` (часто уже как служба).
3. В dev-режиме запросы к модели идут на `/ollama/...` и проксируются Vite на `http://127.0.0.1:11434`.

**Вариант B — внешний OpenAI-совместимый API (например, xAI Grok)**

В `.env` (не коммитьте ключи):

```env
VITE_LLM_COMPAT_URL=https://api.x.ai/v1/chat/completions
VITE_LLM_COMPAT_KEY=ваш_ключ
VITE_LLM_COMPAT_MODEL=grok-2-latest
```

Если заданы `VITE_LLM_COMPAT_URL` и `VITE_LLM_COMPAT_KEY`, клиент использует их вместо Ollama.

## Docker Compose (дополнительно)

```bash
docker compose up --build
```

- API: `http://localhost:8080`
- Клиент (nginx): `http://localhost:5173`

Прокси в контейнере `web`: `/api` → сервис `api`, `/ollama` → Ollama на хосте (`host.docker.internal:11434`). Убедитесь, что Ollama запущена на машине.

## Реализованные требования

- Страницы `/ads`, `/ads/:id`, `/ads/:id/edit` с загрузкой и ошибками
- Список: поиск, сортировка (в т.ч. по цене — поддержано в `server`), фильтры, пагинация по 10, сетка/список
- Просмотр: характеристики по категории, блок «Требуются доработки»
- Редактирование: форма, черновик в `localStorage`, AI «придумать/улучшить описание», «рыночная цена», сравнение «Было → Стало»
- Отмена запросов при уходе со страницы (`AbortController`)
- Дополнительно: чат с AI на странице редактирования, тёмная тема (сохранение в `localStorage`), Docker Compose, юнит-тест (форматирование цены)

## Принятые решения

1. **Поле `id` в списке** — в ТЗ в ответе `GET /items` явно не указан `id`, но для маршрутизации он необходим; в `server` он возвращается.
2. **Сортировка по цене** — в задании упомянута; в mock-сервер добавлен `sortColumn=price` (клиент уже передаёт этот параметр).
3. **Критерий «требует доработок»** — на сервере: пустое описание или неполный набор характеристик для выбранной категории (как в ТЗ для `needsRevision=true`).
4. **Секреты LLM** — ключи только в `.env` (в git не попадают). Для SPA ключ в `VITE_*` всё равно попадает в бандл; для продакшена обычно нужен backend-proxy; для тестового задания допустим явный риск с предупреждением в README.
5. **Визуал** — ориентир на нейтральный «кабинет» и зелёный акцент; пиксель-перфект к Figma не гарантируется без доступа к макетам из среды сборки.

## Публикация в Git

Репозиторий нужно вести **из корня проекта** — папки `avito`, где лежат **`src/`**, **`server/`**, корневой `package.json` и этот `README.md`.

Если выполнить `git init` только внутри **`server/`**, на GitHub попадёт только бэкенд (как у вас сейчас): там **не будет** `src`, потому что фронтенд лежит **на уровень выше** — в `avito/src`, а не в `avito/server/src`.

**Как исправить (типовой вариант):**

1. Удалите вложенный репозиторий: папку **`server/.git`** (или переименуйте её).
2. В корне проекта (`WebstormProjects\avito`):
   - `git init`
   - убедитесь, что в **корневом** `.gitignore` есть `node_modules`, `dist`, `.env` (уже есть в этом шаблоне);
   - в **`server/`** добавлен `server/.gitignore` с `node_modules/` — не коммитьте зависимости сервера из подпапки.
3. `git add .` → `git commit` → привяжите `origin` и сделайте **`git push --force`** на существующий репозиторий GitHub (или создайте новый репозиторий), чтобы заменить историю без `node_modules` и с полным кодом.

**Важно:** не добавляйте в git каталоги **`node_modules`** — они восстанавливаются командами `npm install` в корне и в `server/`.
